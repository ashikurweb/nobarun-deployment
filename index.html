Deployment Documentation

Nobarun Deployment Guide

Complete step-by-step guide for deploying to DigitalOcean

Before You Begin

This guide will walk you through the complete deployment process for the Nobarun project on DigitalOcean. Follow each step carefully to ensure a smooth deployment.

1

Access Your DigitalOcean Account

Log in to your DigitalOcean account and navigate to the domain where you want to deploy the project.

Tip: Make sure you have the necessary permissions to manage droplets and domains.

2

Resize Your Droplet (Optional)

For faster deployment, consider resizing your droplet. There are two approaches:

Option A: Resize Current Droplet

Turn off the droplet first, then resize it for a faster process.

Option B: Switch to Another Droplet

Switch to another optimized droplet size for better performance.

3

Navigate to Project Directory

Access your project's root directory on the server.

Command
cd /home/ploi/bderp.online
4

Reset Git Repository

Discard all local changes and reset to the last committed state.

Command
git reset --hard

Warning: This will permanently discard all uncommitted changes.

5

Pull Latest Changes

Fetch and merge the latest code from the main branch.

Command
git pull origin main

Note: The default branch is 'main'

6

Update NPM Packages

If you've added or updated npm packages, run one of these commands:

Update Packages
npm update
OR
Install Packages
npm install
7

Build Frontend for Production

Compile and optimize your frontend assets for production.

Command
npm run prod

Info: This process may take a few minutes depending on your project size.

8

Update Composer Packages

If you've added composer packages, update your dependencies:

Update Packages
composer update
OR
Install Packages
composer install
9

Run Database Migrations

If there are any new migration files, run the tenant migrations.

Command
php artisan tenants:migrate
10

Clear DomPDF Cache

Remove the DomPDF font cache file from the storage directory.

Command
rm storage/fonts/dompdf_font_family_cache.php
11

Restore Droplet Size

Once the deployment is complete, restore your droplet to its previous size if you resized it in Step 2.

Success: Your deployment is now complete! Your application should be running with the latest changes.

Need Help?

If you encounter any issues during deployment, check the logs or contact your system administrator.
